# Tenchi-MCPアイデアログ.md
名前の由来はクラウドとローカルから、
`クラウド=雲=天`、`ローカル=地域=地`
天地-MCPってこと。
アメリカ軍にちなんでHighLowMixでもいいかなって思ったけど、ローカルLLMだって700Bモデルとかあるし別にそれはLowではないなという判断。
やってることは単純だがクラウドLLMとローカルLLMのハイブリッド構成。
遅くなることは想定済み。元々サブエージェント機能のトークン消費を少しでも抑えたいという意図があります。（全てをデフォルトのサブエージェントでやるとトークン消費が速すぎるので。）

### 私
GeminiCLIってPowerShellでコマンド実行してくれるんだよ、Geminiが。
でさ、もしかしてなんだけど...Ollama入ってるのね、私の環境。それでその...他のモデルに１回推論させればさ、結構変わりそうじゃない？？？いや、速くなるわけじゃないし寧ろ遅くはなるんだけど...コーディング性能上がりそうじゃない...？
面白くね？？クラウドLLMとローカルLLMのハイブリッド構成

どうだろうな、その...んー。いや、正直待ち時間増えるのはもう仕方ないと思ってる。私のPCスペックもどっちかっていうと下から数えたほうが速い、というかほぼ最低限スペックだ。VRAMもあるけど6GBだし。まあ2026年現在ならそれでも9Bクラスのコーディング特化ローカルLLMが普通に動くし、今のGemma4 E4B Q4KMだって、こいつ全部にやらせたら確実に破綻するがあくまで補助なら話は変わる。

なんなら簡易的なアプリにしてMCP鯖化してもいいしな。

MCP鯖つっても大したもんではないよ。
ほらさ、`ollama run gemma4-e4b "ねえなんか面白い数学的命題のコード書いて"`
ってやったら出力してくれるじゃん。
そこをちょっと簡略化してGeminiが呼び出しやすいようにしたらいいのよ。呼び出すモデルはそれこそContinueのconfig.yamlみたいな形式にすればいいでしょ？個人的にはTOML使いたいけど。models_config.tomlって感じでさ。

タイムアウトに関してはそうだな...そこはまた今度決める。
リピートペナルティは強めに設定しないとすぐループしちゃうからそこだけデフォルトで止めないとね。といっても動かしてるのはただのollamaだからCtrl Cで止まる（Geminiが仮にPowerShellを動かしたとしてもユーザーから干渉自体は可能）
言語はそうだな、正直何でもいいとは思う。実際ただollama runしてるだけだからそこ自体には言語関係ないし...私がRust好きなのもあるからRust製かな、クラウドLLMも実質プロンプト添えてexe叩くだけだし。
あー、なんかサーバーっていうと違うんだよなぁ...exeやらのバイナリ置いといてやらせるっていうか...
.gemini\extensions\tenchi-mcp\bin\tenchi-mcp.exe "ここにプロンプトを入力"
を、GeminiCLIやClaudeCodeとかで簡単に出来るってだけ。別にMCPにする必要すらなかったりするんだけどMCPにしておけば再利用が楽ってだけ。
別にローカルホストとか立てる必要もないよ、ollama serveさえ効いてればいいし...

そう、実質なんていうの？私も使えるんだよねｗｗｗ
MCPっていう規格に合わせてCLIのAIが使いやすいようにしただけで。

なんなら出力の無限ループだってあれじゃないの？別にプログラムで監視して止められる気がするんだけど...しかもテンプレチャを低い値にすればループした時同じ文しか出てこないし...
